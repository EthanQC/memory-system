# 个人记忆系统 9 方案横评报告

针对「筑真营销负责人 / Claude Code 深度用户 / 飞书+Obsidian+docs / 不要重向量库 / 分层记忆 / 先个人后团队 / 商业数据敏感」场景。

## 2. Top 3 推荐（按贴合度）

### 推荐 1：Basic Memory（主力）

**为什么最贴合**：唯一同时满足「纯文件 + Obsidian 原生 + MCP 一等公民 + 完全本地 + 零向量库部署」。持久化用 Markdown + frontmatter + `[[WikiLinks]]`，对「先个人后团队」迁移最友好——文件即资产，换工具零损失，飞书/docs 手工粘贴也顺。FastEmbed 本地 ONNX 嵌入，无外发数据。SQLite 仅作索引、可随时重建。

**对主上下文 token 侵占**：MCP 工具约 **12 个**（`write_note` / `read_note` / `edit_note` / `search_notes` / `build_context` / `read_content` / `list_directory` / `move_note` / `delete_note` / `canvas` / `list_memory_projects` 等），按 Claude Code 官方指引估约 **3-5k token 工具定义常驻**。无 always-on 内容注入——完全按需，单次召回典型 2-3 篇笔记约 1-3k token。

**集成概览**：
```
.mcp.json 加一段：basic-memory mcp（uvx）→ 指向 Obsidian vault 目录 → Claude Code 对话中说"记下 XX"即 write_note。
双向同步：basic-memory sync --watch 后台跑。
```

**关键文件**：MCP 入口 `src/basic_memory/mcp/server.py`；工具定义在 `src/basic_memory/mcp/tools/`。

### 推荐 2：Graphiti MCP（时序图谱补强）

**为什么补强**：商业/营销场景天然时序强——「沃林 2026-Q1 客单价 4 万，Q3 上调为 4.5 万」这类事实的**有效期**只有 Graphiti 原生建模（`valid_at` / `invalid_at`，旧事实不删只失效）。配合 Basic Memory 做「结构化客户/项目/合作关系图谱」。FalkorDB 单容器方案部署轻（一个 Docker 而已），MCP 1.0 是官方一等公民。

**token 成本**：HTTP MCP remote 模式，**工具定义约 8 个**（`add_episode` / `search_nodes` / `search_facts` / `get_entity_edge` / `get_episodes` / 3 个删除/状态），约 2-3k token 常驻。

**集成概览**：
```
docker compose（FalkorDB 合一镜像）→ http://localhost:8000/mcp/
本地 embedder 选 Sentence Transformers（非 OpenAI）→ 配置 config.yaml
```

### 推荐 3：MemoryOS（BAI-LAB，分层记忆观察位）

**为什么值得观察**：**唯一不是纯图/纯向量、而是论文驱动的分层 OS**——短期/中期/长期 + user profile 四模块，默认 **JSON+Markdown** 即可跑（ChromaDB 可选），比 Letta 的 Postgres 强制依赖轻得多。MCP 仅 3 个核心工具（`add_memory` / `retrieve_memory` / `get_user_profile`），token 极省（**约 1-1.5k**）。EMNLP 2025 Oral 学术背书。

**建议作为"实验位"**，待稳定后评估是否接管用户画像层。

## 3. 明确劝退名单

- **Mem0（自托管）**：栈三容器；默认外发 OpenAI；SaaS MCP 托第三方违背敏感数据硬约束
- **Letta / MemGPT**：core memory 常驻 2-5k token，违反"不占主上下文"硬约束；围绕 agent 对象建模逆流 Claude Code 形态；Docker+pgvector 强依赖
- **Cognee**：栈最重（向量+图+Postgres）；cognify pipeline 耗大量 LLM 调用
- **A-MEM**：每次 add 都触发 LLM 重写旧 note，成本延迟不可接受；无 MCP、无 UI
- **LangMem**：不在 LangGraph 里用价值砍半；无 MCP
- **Memary**：最后 release 2024-10，半维护状态；相比 Graphiti 全面劣势
- **MemOS**：Neo4j+Qdrant+Redis+SQLite 四件套，个人单用户过度配置
